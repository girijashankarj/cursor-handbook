# Command: Test coverage

Generate or report test coverage.

## Instructions

1. Run the project coverage command (e.g. `npm run test:coverage` or `jest --coverage`).
2. Prefer running for the current file or changed files when possible to save time.
3. Summarize coverage (e.g. statements/branches/functions) and flag lines below project minimum if configured.
4. Do not run the full test suite without user confirmation when token cost is a concern.
