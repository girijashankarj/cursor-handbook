# Prompts

Single-use prompts for common Cursor tasks. Each prompt is self-contained — copy, adjust the placeholders, and paste into Cursor chat.

## Usage

1. Pick the prompt that matches your task
2. Replace `[PLACEHOLDERS]` with your actual values
3. Paste into Cursor Agent or Chat
4. Review and iterate on the output

## Available prompts

| Prompt | When to use |
|--------|-------------|
| [onboarding](onboarding.md) | First day on the project — understand structure and conventions |
| [daily-development](daily-development.md) | Adding features, fixing bugs, writing tests in your daily flow |
| [new-feature-handler](new-feature-handler.md) | Creating a new API endpoint or handler following project patterns |
| [pr-code-review](pr-code-review.md) | Reviewing a PR for correctness, security, and style |
| [testing-quality](testing-quality.md) | Writing tests, improving coverage, fixing flaky tests |
| [security-before-commit](security-before-commit.md) | Pre-commit security check for secrets, PII, and vulnerabilities |
| [troubleshooting-debug](troubleshooting-debug.md) | Debugging a failing test, error, or unexpected behavior |
| [customizing-config](customizing-config.md) | Editing project.json to match your tech stack |
| [replicate-cursor-folder](replicate-cursor-folder.md) | Setting up cursor-handbook in a new project |
| [replicate-docs](replicate-docs.md) | Generating project-specific docs from the handbook templates |
