# Future roadmap

Planned improvements for cursor-handbook (non-binding).

## Short term

- More example `project.json` files for additional stacks.
- Optional flat rule layout (single folder) for teams that prefer it.
- Additional hooks (e.g. post-merge, release tag).

## Medium term

- MCP server examples for internal docs/APIs.
- Pre-built agent packs (e.g. “backend-only”, “frontend-only”).
- Changelog and versioning for the handbook itself.

## Long term

- Web UI for editing rules and agents.
- Metrics on rule/agent usage (opt-in).
- Integration with more IDEs and AI assistants.

Contributions welcome; see `CONTRIBUTING.md`.
